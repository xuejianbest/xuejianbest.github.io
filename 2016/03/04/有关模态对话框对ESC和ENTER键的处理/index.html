有关模态对话框对ESC和ENTER键的处理

有关模态对话框对ESC和ENTER键的处理。

一、关于WM_COMMANDD消息

操作以下对象会发送WM_COMMAND消息：

操作对象 wParam高16位 wParam低16位 lParam
菜单 0 菜单ID 0
加速键 1 加速键ID 0
子窗口控件 控件通知码 控件ID 控件句柄

不同控件发送的控件通知码不一样，如

控件类型 按钮 编辑框
通知码及意义 BN_SETFOCUS 获取焦点 EN_SETFOCUS 获取焦点
BN_KILLFOCUS 失去焦点 EN_KILLFOCUS 失去焦点
BN_CLICKED 单击按钮 EN_CHANGE 内容变化
BN_DBLCLK 双击按钮 EN_HSCROLL 水平滚动
… … … …

举例：

  1. 单击按钮操作：
    系统会向按钮控件父窗口发送消息：
    uMsg == WM_COMMAND wParam == BN_CLICKED*10000H + 按钮ID lParam == 按钮句柄

  2. 选择菜单项：
    系统会像菜单父窗口发送消息：
    uMsg == WM_COMMAND wParam == 0*10000H + 菜单ID lParam == 0

二、关于BS_DEFPUSHBUTTON风格：

作用：

指定此风格的按钮外观不同。若按钮在对话框中，即使按钮没有获得焦点，仍可以默认用ENTER键选择。

补充：

  1. 此风格只对一般按钮有效，对单选钮复选框无效。

  2. 若存在多个按钮被指定此风格，ENTER键只对最后一个按钮有效。

  3. 若没有按钮被指定此风格，但存在ID为IDOK的按钮，则此按钮被指定此风格。

三、关于对话框默认对ENTER和ESC键的处理：

按ESC键：

系统会向对话框过程发送消息：
uMsg == WM_COMMAND wParam == BN_CLICKED*10000H + IDCANCLE lParam == IDCANCEL控件的句柄（假如没有控件ID为IDCANCEL，此参数为0）

四、对话框对关闭操作的默认处理

一般情况下，

  1. 点击右上角关闭按钮
  2. 双击左上角
  3. ALT+F4
  4. 点击左上角，出现菜单后点击关闭

以上操作会引发系统向对话框过程发送WM_CLOSE消息，窗口过程对此消息的默认处理是再向窗口过程发送以下消息：
uMsg == WM_COMMAND wParam == BN_CLICKED*10000H + IDCANCLE lParam == IDCANCEL控件的句柄（假如没有控件ID为IDCANCEL，此参数为0）
由此可知系统对WM_CLOSE消息的默认处理和对ESC键的默认处理是相同的。

用DialogBoxParam创建的模态对话框，在用户对话框过程中无法收到WM_KEYDOWN、WM_KEYUP、WM_CHAR消息。但可以利用上面第三条特性处理ESC和ENTER输入。
